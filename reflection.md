Lecture 7: Libraries and Modular Code
In this lecture, it was shown how libraries help to orangise the program and how to import functions from other files, which helps with the organisation. In my project, I have used this to create separate files rather than writing a large amount of code in one file.
For example, I created a file called student.csv which has student names, IDs, and scores. In another file called filehandler.py, I used the function "import csv" which reads the data in the student.csv file and implements it into this new file. It is quite useful having a separate file for student data, as when I added a few more students, it was easy to locate, and keeps the neat format.
Something else I also used from this lecture was importing functions from other files. For example, I used the function "from filehandler import read_students, write_report", which allows my main.py file to access the functions in filehandler.py. This is useful because I have already defined this functions within the filehandler.py file, so I do not need to rewrite the code, and saves space within the main.py file. 
My code also is similar to the "Command Line Arguments" examples, as in main.py, I have used the 'try', and 'expect' functions on line 10 and 23. What this does is the code within 'try' runs normally and if there are any errors, Python moves to the 'except' part of the code which will handle the problem. Although my code isn't exactly the same as the example, it still uses expectation handling to catch errors and allows the program to keep running.
