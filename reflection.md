Lecture 7: Libraries and Modular Code
In this lecture, it was shown how libraries help to orangise the program and how to import functions from other files, which helps with the organisation. In my project, I have used this to create separate files rather than writing a large amount of code in one file. This is useful because I have already defined this functions within the filehandler.py file, so I do not need to rewrite the code, and saves space within the main.py file. 
I have also applied error handling which includes the "try" and "except" functions, which were shown under "Command Line Arguments". Within main.py on line 10 and 23, it helps prevent crashing as the code within 'try' runs normally and if there are any errors, Python moves to the 'except' part of the code which will handle the problem. Lecture 8: Testing and File I/O
This lecture shows the use of the function "assert" to test functions. It is used to auotmatically verify that functions are working correctly, instead of manually doing so. I have used this shown in test_grades.py where i have written "def test_average(): assert calculated_average([60, 70, 80, 90]) == 75" which checks that the average of those numbers does in fact equal to 75, and if it does it will return with PASSED. If it comes back that it has not passed, you can clearly see what part failed, and can go to the code and fix it. I also used "assert" for letter grade assignment and student ID validation.
I have also shown that I used the File I/O feature, whcih I implemented using the student.csv file. This is demonstrated as my program reads data from this file and processes it to calaculate student overal average scores and their grades. As well as this, the program generates the information into the report.txt file, so you are able to clearly see a table of the IDs, names, average score, and grades, this data is not deleted, so you are able to view it if the program is not running, which can be quite useful.
From workshop 8, I used regex to validate student IDs, which allow the program to check if input follows a specific format. In the grade.py file, i called the function "validate_student_id()" which uses regex to ensure that each student ID follows to correct format. For example, I accidentally used an 'o' instead of '0' for student 18, and it didn't show on the report.txt file, so where the format was incorrect it will dismiss the students information. This is helpful as where I hadn't noticed I wrote the ID incorrect, the program automatically checks for mistakes.
